🧠 Deep Neural Network Summary — Hypertension Risk Classification

📌 Objective

This project developed a deep feed-forward binary classifier in PyTorch to predict whether a patient developed hypertension within a 24-month observational time window. The objective was to apply a grid search across a hyperparameter space to determine the optimal model architecture, using different network depths (n layers), network capacities (n hidden units), batch sizes, hidden unit dropout rates, and early stop patience levels.

The simulated dataset contained 60,000 human patients that were matched with appropriate individual-level confounding variables (i.e. income, age, sex, race, bmi, smoking history, alcohol use, and hypertensive drug usage) as well as the patient's city of residence. Class-stratified sampling was applied to control for class imbalance.

Data were split during preprocessing as follows: 37,500 patients for training, 11,250 patients for validation, and 11,250 patients for holdout testing, which essentially leveraged a 70:30:30 ratio split. All input vectors were temporally stationary (i.e. they were cross-sectional). Numerical input features were standardized to a standard normal distribution, and categorical variables underwent OneHot encoding.

🧱 Model Architecture

Model architectures were defined by a dynamic PyTorch class DynamicBinaryClassifier, which flexibly built a stack of hidden layers using:

  • He (Kaiming) initialization
  • LeakyReLU activations
  • Batch normalization
  • Dropout regularization
  • Raw logits in the output layer
  • Binary crossentropy with logits loss
  • Early Stop: MinDelta (validation loss > 0.001) | Reset Weights

🔧 Architecture Calibration (Grid Search)

We conducted a full grid search over combinations of the following hyperparameters:

  • Hidden layers: 2, 3, 4
  • Hidden units per layer: 128, 256
  • Batch sizes: 128, 256, 512, 1024
  • Dropout rates: 0.2, 0.3
  • Patience for Early Stopping: 11, 13, 15 epochs

📏 McNemar’s Test for Model Disagreement

We used McNemar’s test to compare binary predictions made by the neural network and logistic regression models on the same test dataset. This test specifically assesses whether the two models have significantly different error rates on mismatched predictions, which lets us know if one model is higher performing.

  • McNemar's Chi-Square: 0.2923
  • p-value: 0.5888

Although the optimized neural network yielded a marginally improved ROC AUC (at time of holdout testing), the McNemar's test was not statistically significant at the p<0.05 level. This suggests that the neural network and logistic regression provided similar prediction accuracy and reliability. This is still helpful despite similarity, as we successfully identified a neural net architecture that (i) generalizes well to unseen data and (ii) provides marginally improved prediction capacity versus traditional classification methods (i.e. logistic regression).
